## 📖 项目简介

标书智能检查工具是一款专为招投标行业设计的桌面应用程序，利用大语言模型（LLM）技术自动检查投标文件的合规性和完整性。

### 核心功能

- **📄 智能文件解析** - 支持 DOCX、PDF、XLSX 等多种格式
- **🔍 自动规则提取** - 从招标文件自动提取检查标准
- **🤖 AI 智能检查** - 使用 LLM 进行深度内容分析
- **📊 可视化报告** - 直观展示检查结果和问题
- **🔒 数据安全** - 本地处理，敏感信息自动脱敏

## ✨ 功能特点

### 智能检查能力

| 检查类型 | 说明 | 示例 |
|----------|------|------|
| **完整性检查** | 验证投标文件是否包含所有必需内容 | 必填章节、资质证书、签字盖章 |
| **一致性检查** | 检查项目名称、金额、日期等关键信息一致性 | 报价前后一致性、工期一致性 |
| **合规性检查** | 核对是否满足招标文件要求 | 评分表应答、资质要求、格式规范 |
| **友商标识检测** | 识别文档中的竞争对手名称或 logo | 华为、深信服、奇安信等 |
| **格式规范检查** | 验证文档格式、排版是否符合要求 | 字体、字号、页边距、页眉页脚 |
| **风险识别** | 识别潜在风险点 | 报价异常、承诺过度、错别字 |

### 安全特性

| 安全措施 | 说明 |
|----------|------|
| 🔐 **本地处理** | 所有文档在本地处理，不上传云端 |
| 🎭 **敏感信息脱敏** | 发送给 LLM 前自动脱敏手机、身份证、金额等 |
| 🔑 **API 密钥加密** | 使用 Windows DPAPI 加密存储 API 密钥 |
| 📝 **审计日志** | 记录所有操作，可追溯审计 |
| ⏱️ **速率限制** | 防止 API 滥用，保护账户安全 |
| 🛡️ **文件验证** | 魔数检测、大小限制、内容扫描 |

## 🚀 快速开始

### 系统要求

- Windows 10/11 (64位)
- 最低 4GB 内存
- 200MB 可用磁盘空间

### 方式一：使用编译脚本（推荐）

1. 解压源码包
2. 右键 `build-windows.ps1` → 以管理员身份运行
3. 按提示选择安装路径
4. 等待自动安装依赖并编译

### 方式二：手动编译

#### 1. 安装依赖

**Node.js**
```
下载: https://nodejs.org/
选择: LTS 版本 (v20.x 或更高)
```

**Rust**
```powershell
# 官网下载
https://rustup.rs/

# 或使用 winget
winget install Rustlang.Rustup
```

**Visual Studio Build Tools**
```
下载: https://visualstudio.microsoft.com/zh-hans/visual-cpp-build-tools/
安装时勾选: "使用 C++ 的桌面开发"
```

#### 2. 编译命令

```powershell
# 进入项目目录
cd tender-checker

# 安装前端依赖
npm install

# 开发模式运行
npm run tauri:dev

# 打包发布版本
npm run tauri:build
```

#### 3. 输出文件

```
src-tauri/target/release/
├── tender-checker.exe      # 可执行文件
└── bundle/msi/             # 安装包
```

## 🔧 配置说明

### 配置文件位置

```
%APPDATA%/tender-checker/config.json
```

### 配置示例

```json
{
  "llm": {
    "provider": "baidu",
    "model": "ERNIE-3.5-8K",
    "apiKey": "your-api-key",
    "baseUrl": "https://aip.baidubce.com/rpc/2.0/ai_custom/v1/wenxinworkshop/chat"
  },
  "security": {
    "rateLimit": {
      "hourly": 20,
      "daily": 50
    }
  },
  "desensitize": {
    "enabled": true,
    "rules": ["company_name", "amount", "phone", "id_card"]
  }
}
```

## 🔒 安全最佳实践

### API 密钥管理

1. 使用环境变量或加密存储
2. 不要将 API 密钥提交到版本控制
3. 定期轮换密钥

### 敏感信息处理

1. 所有敏感信息在发送 LLM 前自动脱敏
2. 脱敏映射表仅保存在本地
3. 原始文件不上传云端

### 文件安全

1. 文件类型白名单验证
2. 文件大小限制（默认 50MB）
3. 魔数检测防止伪造扩展名
